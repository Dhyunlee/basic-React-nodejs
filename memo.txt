Git 개요 및 설치
 
 1. Git 개요
     : 버전관리시스템으로 프로젝트 소스를 관리해주거나 협업할 때 사용된다.

 2. 설치
     1) 설치확인:) 명령어: git --version
     2) 없다면: git 홈페이지에서 다운

  3. git 저장소 
     : git 저장소를 생성해주는 역할
     -> 명령어: git init
     - Initialized empty Git repository in D:/DongHyunFolder/따라하며배우는 노드, 리액트시리즈/기본강의/Proects-class/.git/

  4. git 상태
    : git 상태를 알려준다. 
     -> 명령어: git status    

     - Working Directory
        : 작업을 아무것도 안한 상태

    (이렇게 나옴)
  Untracked files:
  (use "git add <file>..." to include in what will be committed)
        index.js
        memo.txt
        models/
        node_modules/
        package-lock.json
        package.json    
  nothing added to commit but untracked files present (use "git add" to track)

     - Staging Area
        1) Staging Area에 올리기(push하기전 push할 파일들을 보관하는 곳)
           -> 명령어: git add .
           -> Staging Area 영역에 node_modules(용량이 너무많음, 나중에 다운로드 가능하므로)
              와 같은 올릴 필요가 없는 폴더들은 제외시킨다.

        2) Staging Area에 올리지 않을 파일 작성
           -> .gitinore 파일을 생성한 후 올리지 않을 폴더명을 작성하고 저장한다.
           -> .gitinore 파일은 git add . 하기전에 작성
              이미 Staging Area 올라갔다면 캐쉬를 제거해준다.
               - 명령어: git rm --cached (캐쉬할폴더명) -r


     - Git repository(local)
        : 로컬 저장소에 올리기 (.git)
        -> 명령어: git commit -m "커밋메세지"
        (컷밋이 완료되면 git status를 입력해서 git 상태 확인)
